# 規定
**ベトナム国家大学ハノイ校におけるオンライン教育の組織・管理および電子講義の構築に関する規定**
*(Giám đốc Đại học Quốc gia Hà Nội が2021年12月29日に公布した決定4391/QĐ-ĐHQGHNに添付)*

# 第I章　一般規定

## 第1条 適用範囲と対象
1. 本規定はオンライン教育について、組織・管理・学習成果の認定、電子教材の構築、関係者の権利と責任、検査・監査・報告、違反処理および実施体制を定める。
2. 適用対象は Đại học Quốc gia Hà Nội（ĐHQGHN）に所属する学部・大学院の教育機関（加盟大学、直属学部、証明書・認定を付与する教育単位を含む）。
3. 中学・高校課程には適用しない。

## 第2条 オンライン教育の目的
1. ĐHQGHN 第6回党大会のデジタル能力指標に掲げるデジタル化目標を達成し、外部学生の正規オンライン科目への受け入れを可能にする。
2. 産業4.0時代におけるICT活用の潮流に対応し、教学・管理・試験評価での情報通信技術と評価ツールを導入する。
3. 自学自習・研究能力を伸ばし、個別化学習のニーズに応え、先進的EdTechプラットフォームによる学術空間を提供する。
4. 正規LMS や MOOC で利用するための標準的な電子講義を構築し、労働市場の需要に適合しつつ内容要件を満たす充実した教材を継続的に更新する。
5. 知識伝達の方法を発展させ、PCや各種端末でのアクセス性を高め、いつでもどこでも自学できるようにする。電子講義は専門内容を満たすだけでなく、学習者が理解・習得しやすい教育設計とする。
6. 対象者を国際的に拡大し、質の高いeラーニング導入を段階的に推進する。

## 第3条 用語の解釈
1. **電子学習（e-Learning）**: 学習者が多媒体の電子教材（講義、ナレーション、音声、画像、動画、グラフィック等）を用いて、いつでもどこでも自学できる形態。m-Learning、u-Learning、smart-Learning なども e-Learning の形態に含まれる。
2. **電子教材（Course-ware）**: 学習成果に沿って構造化・デジタル化された教材で、文書・多媒体を含む。学習者は講師・教育機関が定めた範囲でアクセスする。
    * 2.1. 文書教材: 文書ファイル、スライド、データ表、教科書、専門書、学術誌など。
    * 2.2. 多媒体教材: 動画、音声、インフォグラフィック、アニメーション、またはこれらを統合したインタラクティブな形式。

    多媒体教材のインタラクションには以下の段階がある:
    * 既存のシナリオを起動するためのシナリオ連動型。
    * 入力データに応じてシナリオを生成するタイプ（仮想実験など）。
3. **ブレンデッド学習（Blended learning）**: 第4条1項のオンライン教育システムまたは同条2項の支援システムを活用し、オンライン授業と従来型授業を組み合わせる。対面授業の時間数は総時間数の3分の1以上とする。
4. **電子講義**: LMS や MOOC で提供するために学習者へ効果的に知識・技能を伝えるよう教育的構造でまとめた電子教材群。1講義は1科目に対応する。
5. **電子講義モジュール**: 科目内の1単位の知識に対応する講義の一部。シラバスの章・節や一定の授業時間に相当する。
6. **電子教科書**: 高度に完成した1つまたは複数の電子講義で、学部／学科または ĐHQGHN レベルで専門審査を受けたもの。
7. **電子コース（e-Course）**: 1つまたは複数の電子講義で構成され、特定の学習者集団が科目またはプログラムの学習成果を達成できるよう設計されたもの。講師の支援がある場合とない場合がある。
8. **学習コンテンツ管理システム（LCMS）**: 電子講義を管理し、特定の形式で構造化するソフトウェア。
9. **学習管理システム（LMS）**: コースと学習過程を管理し、学習者・講師間のインタラクションや学習履歴を記録するソフトウェア。
10. **SCORM（Sharable Content Object Reference Model）**: 米国国防総省の ADL が定めるウェブベースのインタラクティブ電子講義に関する規格群。
11. **多肢選択式設問**: 単一選択、複数選択、正誤判定などオンラインで回答できる形式。
12. **オンラインテスト**: 学習者の知識・技能を測るために作成された多肢選択式設問の集合。終了後に総合結果や設問別解答を設定に応じてフィードバックする。
13. **仮想実験・シミュレーション**: コンピュータ上で模擬する実験。データ入力や選択によって操作し、仮想環境で観察・計算を行う。

# 第II章　オンライン教育システムと学習管理

## 第4条 オンライン教育および支援システム
1. オンライン教育システムは、オンライン教育ポータル、LMS、LCMS、電子教材、インタラクション手段、試験・評価、講師活動管理、システム管理などから成る。
2. オンライン教育支援システムは、講師と学習者の授業・インタラクション・評価を実施できるICTコンポーネントの集合である。
3. 支援システムは教育管理ソフトとデータ連携し、時間割や学生・講師情報を同期し、学習結果や講師の授業状況を相互に同期する。
4. MOOC はインターネットを通じて教材を提供する学習形態で、学習者の自己学習・評価に役立て、正規学位課程には適用しない。

## 第5条 学習管理システム
1. 学習管理システム（LMS）は、電子コース、電子講義、適切なインタラクション機能を備えたオンラインテスト、テストシステム、学習者評価、学生の学習結果データの統計分析、講師管理、システム管理、ダッシュボード、および学習者の学習結果を教育ソフトウェアに自動同期する機能を含む。
2. 電子学習教材やデジタル学習教材を多数の学習者に配信し、同時に管理部門が教育プロセスを効果的に監視、調整、評価することを支援する。
3. 学習者のログインアカウントシステムを作成する。
4. 講師の参加を得て、学習について交流し、学習者の質問に答えるための交流フォーラムやその他のサポートツールを作成する。
5. 学習に使用するサーバーシステムとインターネット接続インフラは、ユーザーのアクセスニーズに対応するのに十分な帯域幅と容量を持ち、ネットワークの混雑や過負荷を引き起こさないようにする必要がある。

## 第6条 学習コンテンツ管理システム
1. 学習コンテンツ管理システムは、ĐHQGHNおよび各部門のデジタルライブラリとの同期を可能にし、オンライン学習者に学習コンテンツを提供する。
2. 講師向けの授業準備ツールを統合し、デジタル学習リソースやデジタルライブラリとデータを連携する。
3. 実際の条件に応じて、各部門は学習管理システムと学習コンテンツ管理システムを統合することができる。

## 第7条 電子コースの内容構造
1. 正規のオンライン教育の実施は、教育プログラムの単位時間数の30%を超えてはならない。疫病、自然災害、敵の破壊活動、その他の特別な場合には、ĐHQGHNは現行の規定と実際の状況に適合した個別の指示文書を発行する。
2. 電子コースの構造は、コース名、学習対象者、コースの方向性、時間、電子講義、教科書、一般交流フォーラム、オンラインディスカッション、オンラインテスト、大規模課題/小論文、学習者のフィードバック調査票などの各部分に分かれている。
3. 情報技術の条件に応じて、部門はオンライン教育または統合教育を実施する:
a) 第4条第1項に規定するオンライン教育システムでは、部門は学習単位全体のオンライン教育または統合教育（ブレンデッド・ラーニング）を実施できる。
b) 第4条第2項に規定するオンライン教育支援システムでは、部門は統合教育（ブレンデッド・ラーニング）を実施する。
c) 第4条第4項に規定するオープン学習教材提供システムでは、部門は遠隔教育に使用する情報技術プラットフォームを適用できる（正規教育には適用しない）。
4. 電子講義は、学習単位の教育目標と成果基準に密接に従った内容を持ち、高い教育学的価値があり、使いやすく、基本的なコンピュータスキルを持つ学習者の自己学習ニーズを満たす必要がある。
5. オンラインテスト（中間試験または期末試験）は試験問題バンクから取得し、最低20問を含む。学習単位終了時の多肢選択式試験の時間は最低30分、最長120分とする。
   特定の学習単位の場合、部門長は詳細なシラバスと学習単位の学習成果に基づいて承認する。
6. システムにアップロードされる学習コンテンツは、2013年7月15日付の首相令第72/2013/NĐ-CP号、および2018年3月1日付の首相令第27/2018/NĐ-CP号による修正・補足に従い、インターネットサービスとオンライン情報の管理、提供、使用に関する規定を遵守しなければならない。

## 第8条 電子コースにおけるインタラクション方法
講師は、コンテンツ、期間、学習者数に応じて、適切なインタラクション方法を選択できる：交流フォーラム、個別交流、オンラインディスカッション、電子講義。
1. 交流フォーラムは、講師と学習者、学習者同士が学習コンテンツについてオンラインで交流、ディスカッション、共有できる場所である。
2. 個別交流は、講師と学習者、学習者同士が交流ツール（チャット、電子メール、テキストメッセンジャー、デュオミーティング）を使用して学習コンテンツについてディスカッションし、共有するインタラクション形式である。
3. オンラインディスカッションは、学習者と講師、学習者同士が、電話機器、コンピュータ、またはインターネットに接続された電子機器を介してオンライン教育システムのツールを使用して交流する形式である。この形式を選択する場合、講師はコンテンツ、トピック、実施時間を登録または通知する必要がある。
4. 交流する前に、講師と学習者は、eラーニングシステムの使用規定に従って、ディスカッションするトピックとコンテンツを決定する必要がある。
5. 交流記事の内容は明確で、言葉は明瞭で、フォーラム参加者に対する尊重と適切な態度を示す必要がある。
6. システム管理者は、交流フォーラムで共有されるコンテンツの著作権について責任を負わない。

## 第9条 オンライン教育科目
1. 教育スケジュールに基づき、教育部門長はオンライン教育科目のリストを承認し、部門のウェブサイトまたは教育ポータルで公開する。1学期に多くのオンライン教育科目がある場合、教育部門は一般知識ブロック（M1、M2）に属する科目と専門知識ブロック（M4）に属する補助科目のオンライン教育の実施を優先し、奨励する。
2. 各科目のオンライン教育の内容と期間は講師が作成し、科目のシラバスに示される。
3. 教育部または教育管理部門は、担当対象に応じて教育内容の計画、収集、管理を担当し、学期の時間割で公開し、教育プロセスを監督する。
4. オンラインコースに使用する電子学習教材システムは、コース開始前に学習者に提供する必要があり、オンライン学習プロセスで使用する必須学習教材とその他の自己学習教材（ある場合）を含む。
5. 定期的に、形式と講義内容について見直し、更新、補足、調整を行う（年2回以上）。

## 第10条 教育計画
1. 教育計画は、コースの開始時（コース開始前）から学習者に提供する必要があり、学期には以下が含まれる：コース開始時のオリエンテーション、学習教材、インタラクションのツールと方法、学習者の課題、検査と評価の形式、学習コンテンツの完了期限。
2. 教育部門は、講師、学習者、および機能部門が検査、審査、監督できるように、学期の時間割を部門のウェブサイトで公開する。
3. 正規の教育・学習活動の時間：教育部門の教育・学習活動の時間は、月曜日から土曜日の平日の午前7時から午後8時までとし、ĐHQGHN全体で統一する。教育プログラムの枠組みに従って換算された標準授業時間は、オンライン教育50分、またはオンライン教育とディスカッションの時間を組み合わせたものである。
4. 多くの講師が参加して教育する科目の場合、科目教育計画において主担当講師として割り当てられる講師が1名必要である。

## 第11条 学習結果の評価
1. 学習者の学習結果の評価は、真剣かつ完全に実施しなければならない。評価内容と各種点数の配分は、科目のシラバスとĐHQGHNの現行教育規定で規定される。
2. 評価結果には以下が含まれる：*平常点*（学習者は科目の単位時間の少なくとも80%に参加する必要がある、フォーラムでの交流への参加、評価テスト、課題、セミナーなどの結果）、*中間評価点*（多肢選択式問題への回答、大規模課題/小論文の点数、中間テストなど）、*期末試験点*（オンライン試験または従来の方法で実施される試験、大規模課題、プロジェクト）は科目のシラバスで詳細に規定される。
3. 本規定第4条第1項に規定するオンライン教育システムでは、教育部門は部門長の決定により、オンライン形式または従来の方法で期末試験を実施できる。オンライン試験を実施する場合（ある場合）、多肢選択式試験を作成するのに十分な大きさの問題バンクを持ち、少なくとも4つの独立した重複しない試験問題の作成を保証する必要がある。部門長は、オンライン試験実施プロセスのガイダンス文書を発行し、試験問題、解答、採点ガイドは機密保持されなければならない（試験前、試験中、試験後）。
4. 本規定第4条第2項に記載されているオンライン教育支援システムでは、教育部門はĐHQGHNの現行規定に規定されているオンライン形式または従来の形式で学習単位終了試験を実施する。
5. 本規定第4条第4項に規定するオープン学習教材提供システムでは、部門は遠隔教育結果の証明書を発行でき、ĐHQGHNの正規学位授与教育プログラムには適用しない。
6. 実技スキルを評価する実習/実技科目（技能試験、業務、技術操作など）では、オンライン形式で期末試験を実施しない。特別な場合は、科目のシラバスと教育プログラムの成果基準の詳細な説明に基づいて部門長が決定する。
7. 自然災害、敵の破壊活動、疫病、不可抗力の場合、教育部門が学習者に直接防御のために組織できない場合、オンライン防御を組織できる：卒業論文、修士論文、博士論文セミナー。特別な場合はĐHQGHN長官が決定する。

# 第III章　電子講義構築のプロセス

## 第12条 電子講義のレベル
電子講義は IT を活用したマルチメディア環境で授業内容・活動をデジタル化する。情報はテキスト、グラフィック、アニメーション、画像、音声、動画、仮想実験、3D、VR、AR 等で提示する。レベルは以下のとおり:
* **レベル1（S: Substitution）**: スライドや動画等を用いた提示中心で、講義を補助・代替する。
* **レベル2（A: Augmentation）**: 画像・シミュレーション等の補足情報を含み、画面上でのインタラクションや仮想実験、チェックツールにより学習者の主体性を高める。
* **レベル3（M: Modification）**: 動画・アニメーション・音声・スライド等の多媒体を教育的に構成し、学習者が選択・演習・自己評価を通じて自学できる。
* **レベル4（R: Redefinition）**: 仮想空間、3D、360°動画、VR/AR/MR により新たな学習活動と機能を創出する。

レベル3以上は自学可能で、電子教科書と見なせる。

## 第13条 電子講義の評価基準
電子講義は以下を満たすこと。
**1. 内容**
* 科目の学習成果とプログラムの成果マトリクスを満たす知識・技能・態度・倫理を提供する。
* 重点内容を明確にし、教育的価値の高い実践的関連付けを行う。
* 誤りなく体系的で、承認されたシラバスの目標に適合する。
* 党の方針や国家法令に反する内容を含まない。
* 参考文献・資料の出典を明確にする。

**2. 教育性・伝達方法**
* 問題提起―知識形成―演習―まとめ―体系化―評価の各段階を備える。
* 学習者に適し、魅力的で分かりやすく受け入れやすい。
* 学習状況を設定し、学習者を導き、問題を提起・解決できるようにする。

**3. 技術・工学**

## II. 現行 eラーニング規格
eラーニングシステムは、すべての制作、学習、管理、またはシステム内のコンポーネントとのインタラクションプロセスを統一するために、対応する規格を適用している。規格グループには以下が含まれる：
- **第1グループ：** 異なるメーカーの異なるツールで作成された学習教材をコンテンツパッケージ（packages）として結合することを可能にする。この規格グループはパッケージング規格（packaging standards）と呼ばれる。このシステムのおかげで、学習管理システムは異なる学習教材をインポートして使用することができる。
- **第2グループ：** 学習管理システムが各レッスンを表示し、テスト結果や各学習者の学習プロセスを追跡することを可能にする。この規格グループは情報交換規格（communication standards）と呼ばれる。
- **第3グループ：** 学習管理システムが必要に応じて検索および分類できるように、学習教材の内容を記述することを可能にする。この規格グループはメタデータ規格（metadata standards）と呼ばれる。
- **第4グループ：** 学習教材の設計プロセス全体および学習者への学習教材の支援能力、障害者への補助を制御することを可能にする。この規格グループは品質規格（quality standards）である。

**1. パッケージング規格**
パッケージング規格グループは、個別の学習オブジェクトを組み立てて電子教科書や独立したコンテンツ単位を作成し、その後、多くの異なるLMS/LCMSシステムで転送および再利用できるようにする方法を記述する。
以下は、講義内容のパッケージング規格セットの一部である：

| 規格セット | コメント |
| :--- | :--- |
| **AICC**<br>(Aviation Industry CBT Committee) | - 学習教材がインタラクションできることを保証するために、AICC仕様は複雑さのレベルに応じて多くのファイルを必要とする。具体的には、学習教材記述ファイル、他のコンテンツ単位、記述ファイル、学習教材構造ファイル、条件ファイルなどを含む。<br>- この仕様は、コンテンツの複雑な構造を設計することを可能にする。 |
| **IMS Global Consortium** | - IMS Content and Packaging仕様はAICCよりも単純であり、eラーニングコミュニティによって広く受け入れられ、適用されている。Microsoft LRN Toolkitなどの一部のツールソフトウェアは、この仕様の実装をサポートしている。 |
| **SCORM**<br>(Sharable Content Object Reference Model) | - SCORMは、IMS Content and Packagingを含む多くの異なる仕様を組み合わせている。<br>- SCORM 2004では、IMSのSimple Sequencing 1.0仕様が追加された。現在、大多数のeラーニング製品はSCORMをサポートしており、最も注目すべき仕様である。 |

**2. 情報交換規格**
情報交換規格は、人間またはコンピュータが互いに情報を交換するために使用するプロトコルまたは言語を定義する。情報交換規格の一例は、一般的な単語を定義する辞書である。eラーニングでは、情報交換規格は、学習管理システムがコンテンツユニットと情報を交換できる言語を定義する。

学習管理システムと学習オブジェクトの間で交換される情報の内容には、以下のようないくつかのトピックが含まれる：
- 管理システムは、学習オブジェクトがいつ動作を開始するかを知る必要がある。オブジェクトは学習者の名前を知る必要がある。
- オブジェクトは逆に、学習者がオブジェクトの学習を何パーセント完了したかを管理システムに通知する。
- 管理システムは、データベースに保存するために、学習者が取得したスコアに関する情報を知る必要がある。
- 管理システムは、学習者がいつ学習を終了し、学習オブジェクトを閉じるかを知る必要がある。

情報交換規格には、主にプロトコルとデータモデルの2つの部分が含まれる。プロトコルは、管理システムと学習オブジェクトが互いに情報を交換するプロセスを定義する。データモデルは、テストのスコア、学習者の名前、学習者のレッスン完了レベルなど、交換プロセスで使用されるデータ構造を定義する。

**3. メタデータ規格**
eラーニングにおけるメタデータ（metadata）規格は、学習教材と学習オブジェクトに関する記述データである。メタデータ規格は、さまざまな種類のeラーニングを記述する方法を提供する。

*3.1. メタデータ規格の役割：*
- メタデータ規格は、複雑で制限のないフレーズの検索を実行することを可能にする。特定のトピックについて他の言語で学習教材を検索したり、その言語のドキュメントの内容全体を閲覧することなく、あらゆるドキュメントを検索したりすることもできる。
- メタデータはまた、学習教材、レッスン、およびその他のコンポーネントを分類することを可能にする。作成者が必要なコンテンツを見つけて使用するのに役立つ。

*3.2. いくつかの現行メタデータ規格：*
実際に適用され標準化されているメタデータ規格には3つの仕様がある：
- Learning Technology Standards Committee of IEEE: Learning Object Metadata Standard IEEE 1484.12.
- IMS Global Learning Consortium: Learning Resources Metadata Specification.
- ADL: SCORM Metadata standards.

*3.3. メタデータの基本コンポーネント：*
IEEE 1484.12規格（つまりLOM）は、すべての標準化団体によって認識され使用されているため、共通規格と見なされている。以下は、IEEE 1484.12規格に属する（主な）コンポーネントのリストである：
- **Title:** 学習教材の正式名称。
- **Language:** 学習教材内で使用される言語を特定し、追加情報を含む場合がある。
- **Description:** 学習教材の説明。
- **Keyword:** 検索をサポートするキーワードを含む。
- **Structure:** 順序や階層など、学習教材の内部構造を記述する。
- **Aggregation Level:** ユニットの結合レベルを特定する：4 – 学習教材、3 – レッスン、2 – 項目。
- **Version:** 学習教材のバージョンを特定する。
- **Format:** 学習教材で使用されるファイル形式を規定し、MIME規格の形式と互換性がある。
- **Size:** 学習教材に含まれるすべてのファイルの合計サイズ。
- **Location:** 学習教材にアクセスできるWebアドレス。
- **Requirement:** 学習教材を実行するために必要なブラウザやオペレーティングシステムなどの条件をリストする。
- **Duration:** 学習教材を学習するための平均時間を特定する。
- **Cost:** 学習教材が無料か有料かなどを示す。
- メタデータ規格に準拠する一部のツール：情報システム間のインタラクションを保証するためにXML形式を使用する必要がある。

**4. 品質規格**
品質規格は、学習教材の設計と学習者、および障害者が学習教材にアクセスできる能力に関連している。
品質規格は、作成されたコンテンツと学習者がそのコンテンツを容易に受け入れられることを保証する。
品質規格は、学習教材が再利用可能であるだけでなく、最初の学習からすぐに使用できることを保証する。

eラーニング設計規格：eラーニングの主な設計品質規格は、ECI（E-Learning Certification Institute）組織のe-Learning Courseware Certification Standardsである。ECI研究所は、eラーニング学習教材が、インターフェース設計、オペレーティングシステムや標準ツールとの互換性、および学習教材の制作と設計の品質など、特定の規格に準拠していることを証明する。

アクセシビリティ：視覚障害者、難聴者、または目と手の協調が良くない人など、障害者が情報技術製品にアクセスできるようにするにはどうすればよいかに関連している。

**5. その他のE-Learning規格**
*5.1. IMSのeラーニング規格*
IMS組織によって編集され、実際に普及しているeラーニング規格。IMSのいくつかの規格：
- **IMS Question and Test Interoperability**は、「質問とテストの相互運用性」に関する規格である。この仕様は、質問とテストを多くの異なる情報システムまたはソフトウェアで使用できるようにする方法を見つけるためにIMSによって提案された。
- **IMS Enterprise Information Model:** 学習管理システム、学習コンテンツ管理、または仮想学校は、組織、機関、または企業の他のシステムと情報を交換できる必要がある。「企業情報モデル」仕様は、異なる情報システムまたはソフトウェア間で管理データを交換できるようにする共通の形式を定義するためにIMSによって提案された。
- **IMS Learner Information Packaging:** IMSの「学習者情報のパッケージ化」仕様は、学習者情報に関する共通の形式を定義するために提案された。この仕様に準拠した記述は、異なる情報システムまたはソフトウェア間で簡単に交換できる。
- **IMS Digital Repositories:** eラーニングに属するコンポーネントを含むデータベースに関するIMSの仕様。
- **IMS Simple Sequencing:** IMSの仕様で、現在はSCORM 2004規格セットに含まれており、ユーザーが許可された「単純なシーケンス」を作成できるようにすることを目的としている。
- **IMS ePortfolio:** IMSの標準的な「履歴書」形式に関する仕様。

*5.2. 通信規格*
通信規格はインターネット上で適用され、eラーニングにも適用される。このような規格は、遠隔地からのリンクと情報交換を目的としてさまざまなツールを組み合わせる場合に必要である。いくつかの現行通信規格：
- **H.323:** パケットベースのマルチメディア情報交換システムに使用される。インターネットプロトコル（IP）ネットワークを介したビデオによる遠隔会議の送信における互換性を強化する。
- **T.120:** マルチメディア会議用のデータプロトコルに使用される。これには、オンライン会議（online-meetings）の会議およびアプリケーション共有に関するプロトコル文書が含まれる。

*5.3. メディアに関する規格*
メディアに関する規格は、メディア情報の標準ファイル形式を規定する。この種の規格の大部分は、World Wide Web Consortium（W3C組織）に由来する。以下は、eラーニングにおける一般的なメディアに関するいくつかの規格である：
- **CSS (Cascading Style Sheet):** 「スタイルシート」。HTMLおよびXMLページの外部インターフェースを制御するために使用される。
- **DOM (Document Object Model):** 「ドキュメントオブジェクトモデル」。ブラウザおよびWebページをプログラムするために使用される。

# 付録III　電子学習教材の内容構造
*(決定4391/QĐ-ĐHQGHN（2021年12月29日、ベトナム国家大学ハノイ校長官発行）に添付)*

## I. 学習教材の構成要素
完全な学習教材を構成する8つの要素は以下の通り：
1. 学習単位の構成（形式）
2. 一般情報
3. 質問
4. 自己テスト（self-test）
5. 理論的知識
6. 用語、キーワード
7. 演習問題
8. 概要、シラバス

実際には、特定の学習単位によっては、自己テスト、用語、キーワード、演習問題（4、6、7）などの補助的な要素が不要な場合がある。残りの要素（1、2、3、5、8）は最小限必要であり、学習教材に必ず含まれなければならない。

## II. 学習教材の一般要件
**1. シナリオの準備**
- 内容の準備：集合N
- プログラム化方式による知識の細分化：集合N1、N2、...、Nkおよび対応する質問Q1、Q2、...、Qk
- 教師と学習者の活動集合の構築：集合T1、T2、...、TkおよびH1、H2、...、Hk
- 結果としてシナリオ{Mi}i=1..kを得る

**2. シナリオの構築**

**表1. 学習教材シナリオの構築**

| :--- |
| **テーマ、講義の題目:** |
| **章・課内の画面ページ番号:** |

| 番号 | 画像内容 | 講義内容 | 予定時間 | 画像効果 | 画面説明 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| | | | | | |

**3. 学習教材の必要基準**
- 学習目標を明確に示す
- コース参加時の前提条件を示す
- 学習教材の内容に関する要約説明情報を含む
- 明確で論理的な構造を持つ
- 学習目標に適合した正確な内容を持つ
- 学習者にどこから学習を開始し、どのように学習を進め、どのような条件で学習するかを指導する
- 学習者が具体的な活動を通じて学習する
- 知識内容と学習者の間のインタラクションを保証し、学習者が典型的なスキルを形成できるようにする
- 各知識内容について、参考資料、知識強化演習、スキル向上演習を含む
- 多様で豊富かつ合理的な学習リソースを持つ

**4. 電子学習教材の評価基準**
- 使いやすく、学習内容を閲覧しやすい親しみやすいインターフェース
- 電子学習教材と他の伝統的な学習形式との学習関係を示す
- 積極的で現代的な多様な教育方法を統合することで教育方法を改善し、コース参加者の積極性、主体性、創造性を最大限に発揮させる
- これにより、学習者は学習過程における自身の進歩の程度を自己評価できる
- 学習者が知識内容の受容を完了した後、応用演習や発展演習を完成できるよう支援する

## III. 学習教材の構造
学習教材の構造を表現する方法は多数あるが、学習教材の構造は以下の部分で構成されると考えることができる：

**1. コースに関する一般情報:**
コースに関する基本情報を示す。これらの情報は、学習者がコース開始時に最初に参照するもので、既知の知識を省略し、授業の新しい知識内容に集中できるようにする。以下を含む：
- コース名
- 作成者
- 単位数
- 目標
- 要約内容
- 前提条件
- 学習単位の評価
- 学習教材の構造
- 学習活動

**2. 学習ガイド資料**
学習者の自己学習を保証するため、学習ガイド部分の内容は以下を含む：
- 紹介、各学習内容を移動する方法
- いくつかの学習活動の具体的なガイド
- 学習計画に関する情報

**3. コース内容**
これは電子学習教材の主要内容である。通常、各トピックごとの知識内容のディレクトリツリー形式で表現される。

**4. 一般参考資料**
- 印刷資料
- オンライン資料

# 付録VI　電子教材に関する規定
*(決定4391/QĐ-ĐHQGHN（2021年12月29日）添付)*

## I. 電子教材の構成
電子教材は以下の4要素で構成し、目安時間を示す:
1. **Text**: 科目ガイド約10〜15ページ（自学45分相当）。Times New Roman 13pt、行間1.2。
2. **学習課題**: 自由記述・ケース・実習等（自学90分相当）。各講義につき最低5問、詳細な実施手順を付す。
3. **Rich Media**: 動画・音声・スライド・自動採点付き多肢選択・参考文献リスト・スライド注記を含む。SCORM 形式で出力する。
    *各講義で、導入、目標、必要知識、参考文献、主要内容、講義内容を示す。
    *Presenter には講師の写真または動画を推奨。
    *各講義に少なくとも20問の多肢選択式設問を用意し、後日追加可能とする。
    *すべての講義で統一テンプレートを用いる。最終スライドでは全体の要点を簡潔に要約し、復習しやすくする。
4. **Audio**: 講義音声。
    *スライド音声を流用せず新たに録音する。
    *朗読は明瞭で抑揚を付け、速度を適切にする。
    *BGM は小さめで集中を妨げないものとする。
    *音質160/320 kbps、形式 mp3。

## II. 電子教材制作契約
任務に基づき、教育機関は指名された責任者または講師等と制作契約を締結し、責任者が計画通り教材を構築する。

## III. 内容審査
責任者は講師・専門家を招き、シナリオ、テキスト、スライド、多肢選択と答案、論述問題、演習を作成する。単位長は内容審査委員会を設置し、審査記録に基づき計画どおり制作を進める。

## IV. 技術審査
教材完成後、責任者はすべての成果物と資料を教育機関に提出し、技術審査を受ける。資料には内容審査記録・評価コメントと、電子教材の検収申請書を含む。教育機関は構成と技術基準の審査を行う。
